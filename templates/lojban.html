Lojban as an Interlingual

Background Information

This is undergraduate research I'm doing for OSCAR. Currently, when Google Translate does translation, it does so by translating to English and then to its target language. English, being chosen as the "Interlingual" or intermediate language for translation, is simply by the fact of being the lingua franca of many modern industries and the best well studied language. However, it, like many languages, contain ambiguities built into the language that can cause inaccuracies during translation. This is unlike constructed languages, in particularly, like Lojban, whose purpose was to provide a means of communicating unambiguously for use in diplomacy and the expression of logic. The goal is to determine whether Lojban's unambiguous properties can produce objectively better translations than that of Google Translate.

Source evaluation
  • Camxes
    • Python module which parses Lojban and turns the text into trees, JSON files, etc. It essentially works as an automatic grammar checker.
  • Camxes-Py
    • Developed by Riley Martinez-Lynch. A different parser, with more options to change things like rules, serialization, and transformation. Pure Python.
  • Valis
    • A postgres database with API for dynamic dictionary search for the meaning of lojban words. It is an updated version of jbovlaste, which is quite old.
  • jbokrop
    • A search engine for text within the corpora. The text are stored in xml files.
  • jbofihe
    • A bundle of software. It does checks grammar of a lojban text, displays grammar trees, produces literal translations.
  • ilmentufa
    • A Syntactic checking for lojban. Specifically developed for la bangu style teaching (A simplified lojban).
